Sidelink Primary Synchronization Signal
########################################
The sidelink primary synchronization signal (S-PSS) is a m-sequence of length 127 generated using :math:`N_{ID,2}^{SL}`.
It is used for time synchronization which aims to find the slot boundary by the user equipment (UE). The :math:`N_{ID,2}^{SL}`
provides a cyclic shift to a base sequence as defined in [3GPPTS38211_S_PSS]_.

..  important::
    * S-PSS is an constituent of sidelink synchronization signal block (S-SSB). It is loaded in the middle 127 subcarrier (across frequency) of :math:`2^\text{th}` and :math:`3^\text{th}` symbol of the S-SSB.

The sidelink primary synchronization signal is generated as follows:

**If** :math:`N_{ID,2}^{SL}` **is scalar:**

.. code-block:: Python

    nID2         = 1           # Cell ID-2 for PSS generation
    spss         = S_PSS(nID2) # Generate PSS object using this cell ID-2.
    spssSequence = spss()      # Generate [1x127] pss sequence using 'pss' object

**If** :math:`N_{ID,2}^{SL}` **is NumPy array:**

.. code-block:: Python

    nID2         = np.array([1,0,1,1]) # Cell ID-2 for PSS generation
    spss         = S_PSS(nID2)         # Generate PSS object using this cell ID-2.
    spssSequence = spss()              # Generate [4x127] pss sequence using 'pss' object for each cell-ID2
